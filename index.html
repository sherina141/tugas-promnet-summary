Version Control System

Pengertian

Version Control Sistem adalah untuk sistem yang biasanya digunakan untuk mencatat, mengelola, dan mengontrol setiap perubahan pada file atau kode program dan dengan version control system pengguna dapat melihat Riwayat perubahan. Dan Kembali ke versi sebelumnya, serta memudahkan kerja tim dalam kolaborasi tanpa khawatir file tertimpa atau hilang.

Jenis-Jenis

Lokal Version Control Sistem adalah sistem yang menyimpan versi file hanya di computer lokal. Perubahan dicatat secara sederhana sehingga mudah digunakan, hanya saja rawan hilang jika data computer rusak. Centralized Version Control System (CVCS) adalah system dengan satu server pusat tempat semua file dan Riwayat tersimpan. Cocok untuk kerja tim, tetapi sangat bergantung pada server. Distributed Version Control System (DVCS) ialah system dimana setiap pengguna memiliki Salinan pernuh repository beserta riwayatnya. Dapat mengerjakan dengan offline, lebih aman, dan juga fleksibel untuk kolaborasi.

Git dan GitHub

1. Git ialah sistem control version yang digunakan untuk melacak perubahan pada kode atau suatu file. Dengan git juga, kita dapat menyimpan versi berbeda dari proyek yang kamu buat. Membandingkan perubahan dan kembali ke versi ke versi sebelumnya jika diperlukan.
2. GitHub ialah platform berbasis web yang dapat digunakan untuk menyimpan repositori di Git secara online. Selain menyimpan, GitHub juga memudahkan kolaborasi derngan orang lain di seluruh dunia.

Perintah dari GitHub

- Git init adalah membuat repository Git lalu di folder lokal.
- Git clone ialah mengunduh repository dari GitHub ke computer lokal.
- Git status ialah mengecek status file apa yang sudah diubah dan sudah ditambahkan ke staging, atau belum nya.
- Git add adalah menambahkan file ke staging area sebelum commit dan berarti semua file.
- Git commit -m ialah menyimpan perubahan dari lokal ke GitHub ( repository online ).
- Git push / Git push -u mengirim perubahandari lokal ke GitHub.
- Git pull adalah mengambil update terbaru dari GitHub ke repository lokal.
- Git log ialah melihat Riwayat commit yang sudah dilakukan.
- Git branch melihat daftar branch atau membuat branch baru.
- Git checkout ( nama branch ) yaitu pindah ke branch tertentu.

Fungsi

Tracking Perubahan

Melacak perubahan atau perkembangan suatu data, dokumen, atau objek dari waktu ke waktu.

Version Management

Mengelola berbagai versi dari suatu dokumen, file, atau perangkat lunak agar perubahan dapat dicatat dan dikontrol.

Data Protection

menjaga file tetap aman karena riwayat tersimpan rapi dan bisa dipulihkan.

Branching dan Merging

memungkinkan pengembangan paralel di cabang berbeda lalu digabungkan kembali.
